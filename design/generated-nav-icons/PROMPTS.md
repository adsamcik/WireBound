# Generated Navigation Icon Prompts

These six sidebar icons were generated individually with the built-in image generator, then converted from the generated chroma-key PNGs into transparent RGBA PNGs.

Shared design system:

- Asset type: standalone UI navigation icon for the WireBound desktop app sidebar.
- Background instruction: render on a perfectly flat solid `#00ff00` chroma-key background for local background removal.
- Style: clean modern app icon, simple silhouette, warm ivory `#F7F0D8`, rounded caps and joins, chunky enough to read at 20-24 px.
- Constraints: no text, no labels, no numbers, no watermark, no UI panel, no circle badge, no border, no shadows, no gradients, no glow, no 3D, no color accents.
- Composition: centered square canvas with generous padding and no clipping.

## Overview

Create one carefully designed Overview icon only: a simple, highly recognizable dashboard/speedometer glyph with a bold semicircular gauge arc, three short tick marks, and a clear needle pointing up-right from the lower center. It must read as "overview / dashboard" at very small sidebar sizes. The glyph should occupy about 70% of the canvas width and 58% of the canvas height.

## Live Chart

Create one carefully designed Live Chart icon only: a bold L-shaped chart axis at the lower-left and a rising zig-zag trend line with three clear segments ending upward on the right. It must read as "live chart / analytics" at very small sidebar sizes. The glyph should occupy about 66% of the canvas width and 62% of the canvas height.

## Apps

Create one carefully designed Apps icon only: four rounded square app tiles arranged in a 2x2 grid with even spacing. Each tile should use a thick rounded outline or solid chunky tile shape with a small transparent center cutout only if it improves readability. It must read as "apps / application grid" at very small sidebar sizes. The grid should occupy about 56% of the canvas width and height.

## Connections

Create one carefully designed Connections icon only: three bold circular endpoint nodes connected by thick rounded lines, with one left node connected to a right node and a third node branching upward-right or downward-right. It must read as "connections / linked devices / network topology" at very small sidebar sizes, not eyeglasses or a dumbbell. The network should occupy about 64% of the canvas width and 52% of the canvas height.

## System

Create one carefully designed System icon only: a bold rounded-square microchip body with a smaller inner square core and short symmetric pins on all four sides. It must read as "system / CPU / hardware" at very small sidebar sizes. The chip should occupy about 58% of the canvas width and height.

## Settings

Create one carefully designed Settings icon only: three horizontal slider rails with bold circular or rounded pill knobs at different positions, specifically top knob left, middle knob right, and bottom knob center-left. It must read as "settings / controls / tuning" at very small sidebar sizes. The sliders should occupy about 66% of the canvas width and 52% of the canvas height.
